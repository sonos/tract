# tract-coreml — Op coverage matrix

This document accompanies the **tract-coreml** PR. It lists:
1. The canary corpus of models that drove translator development.
2. The complete op-family → MIL-op mapping for each translator.
3. Op classes that don't yet have a translator (gaps).
4. Op classes Apple's side has but we deliberately don't emit.

Source of truth in the repo: `coreml/docs/pr-prep.md` §§2–3.

---

## 2. Models tested (canary corpus)

13 model classes probed across CNN / segmentation / transformer / object-detection / super-resolution / audio / decoder-LLM. Each canary lives at `tract/coreml/tests/<model>_e2e.rs` with `#[ignore]`'d `*_coverage_probe` and (where applicable) `*_perf_*` tests.

**Source-of-truth perf matrix lives in [`pr-prep-perf-matrix.md`](./pr-prep-perf-matrix.md)** — generated by a measurement run (CoremlOp counts, first-run latency, steady-state RTF, CPU baseline where available, ANE engagement). The summary below is the structural picture.

### Summary stats

- **13 model classes / 16 distinct probes**, covering CNN classifier / segmentation / video matting / object detection / SR / transformer encoder / windowed-attention transformer / NNEF audio / decoder LLM

## 3. Op coverage matrix (translators)

28 translators across `tract/coreml/src/ops/*.rs` (excluding `mod.rs` and `rank.rs`-helper). Each handles one tract op family or one MIL op family.

### 3.1 Translators by file

| Translator file | tract op(s) consumed | MIL op(s) emitted | Notes |
|---|---|---|---|
| `conv.rs` | `Conv` | `mb.conv` | 2D conv; NCHW + CHW data formats; `SameUpper`/`Explicit`/`ExplicitOnnxPool(ceil_mode)`/`Valid` padding; group ≥ 1; OIHW kernel. |
| `binop.rs` | `TypedBinOp(Add\|Mul\|Max\|Min\|Sub\|Div\|CompEq)` | `mb.add` / `mb.mul` / `mb.maximum` / `mb.minimum` / `mb.sub` / `mb.real_div` / `mb.equal` | Const-or-Data inputs (BinOpInput enum); rank-4 broadcast; bool output for `Eq`. |
| `cast.rs` | `Cast` | `mb.cast` | F32↔F16, I64→I32, etc. Walks `Cast(Const)` chains for absorption. |
| `concat.rs` | `TypedConcat` | `mb.concat` | Variadic inputs, Const-or-Data per slot (CLS-token in ViT), defensive rank-4 reshape. |
| `maxpool.rs` | `MaxPool` | `mb.max_pool` | Same padding/ceil_mode handling as `conv`. |
| `avgpool.rs` | `SumPool(normalize=true\|false)` | `mb.avg_pool` (× area for non-normalized) | `normalize=true` → AvgPool; `normalize=false` → AvgPool × kernel area. |
| `reduce.rs` | `Reduce<Sum\|Max\|Min\|MeanOfSquares>` | `mb.reduce_sum` / `mb.reduce_max` / `mb.reduce_min` / `mb.reduce_mean` (over square) | Multi-axis reduction with keep_dims. |
| `softmax.rs` | `Softmax`, `LogSoftmax` | `mb.softmax`, `mb.log_softmax` | Single-axis. |
| `activation.rs` | `ElementWiseOp` (15+ ops) | `mb.<name>` | Sigmoid, Tanh, Silu, HardSwish, GELU (TANH_APPROXIMATION + EXACT), Erf, Rsqrt, Sqrt, Square, Ln→log, Recip→inverse, Cos, Sin, Neg→mul-by-minus-one, LeakyRelu (with α). |
| `slice.rs` | `Slice` | `mb.slice_by_index` | Single-axis start..end. Defensive rank-4 reshape. |
| `add_axis.rs` | `AxisOp::Add` | `mb.expand_dims` | Inserts unit dim at `axis`. |
| `rm_axis.rs` | `AxisOp::Rm` | `mb.squeeze` | Removes unit dim at `axis`. |
| `reshape.rs` | `AxisOp::Reshape(at, from, to)` | `mb.reshape` | External-shape boundary stripping for rank > 5. |
| `move_axis.rs` | `AxisOp::Move(from, to)` | `mb.transpose` | Single-axis move via permutation. |
| `pad.rs` | `Pad(constant\|reflect\|replicate)` | `mb.pad` | All modes; constant_val required. |
| `resize.rs` | `Resize` (from `tract-onnx-opl`) | `mb.upsample_nearest_neighbor` / `mb.upsample_bilinear` | nearest + bilinear modes. |
| `broadcast.rs` | `MultiBroadcastTo` (= ONNX `Expand`) | `mb.tile` | NumPy-style broadcast. |
| `gather.rs` | `Gather` | `mb.gather` | Embedding lookup + ViT CLS pluck. Const-or-Data per slot. |
| `iff.rs` | `Iff` (3-input cond/t/f) | `mb.select` | Bool cond + matching-dtype t/f. |
| `scatter_nd.rs` | `ScatterNd(reduction)` | `mb.scatter_nd(mode)` | All 5 reduction modes (None/Add/Mul/Min/Max). |
| `instance_norm.rs` | Multi-node InstanceNorm chain pattern | `mb.instance_norm` | Pattern detector — anchors at final Mul, walks back to verify the 6-op chain (ReduceSum + Mul(1/N) + Sub + Reduce<MeanOfSquares> + Add(eps) + Rsqrt + Mul). |
| `layer_norm.rs` | Multi-node LayerNorm chain pattern | `mb.layer_norm` | Pattern detector — recognises the 6-op LayerNorm chain (ReduceSum + Mul + Sub + RmsNorm + Mul(γ) + Add(β)). |
| `rms_norm.rs` | `RmsNorm` (standalone) | `square + reduce_mean + add(eps) + rsqrt + mul` chain | Lowers to MIL primitives (no `mb.rms_norm` exists). |
| `einsum.rs` | `EinSum` (1×1-conv-shaped: 5 axes patterns) | `mb.conv` (1×1) | Maps `IHW,OI->OHW`, `NIHW,OI->OHW`, etc. to 1×1 conv. |
| `matmul.rs` | `EinSum` (matmul-shaped, 4 specific patterns) | `mb.matmul` (with reshapes) | `mkba,kn->n`, `k,kn->mnab`, `IHW,OI->O`, `I,OI->OHW`. |
| `general_matmul.rs` | `EinSum` (general) | `mb.matmul` (with pre/post transposes + reshapes) | Multi-axis M/N/K, batch axes at differing positions, strip-A/B/expand-out unit dims, runtime + const operands. **External-shape boundary mechanism** for rank > 5 SAM 2 windowed-attention forms. |
| `einsum_outer.rs` | `EinSum` (no-K outer-product) | `mb.mul` with broadcast | RoPE patterns like `m,an->abnm`. |

### 3.2 Cross-cutting infrastructure

| File | Role |
|---|---|
| `mod.rs` | Module registry + shared `const_tensor` helper that walks `Cast(Const)` chains. |
| `rank.rs` | `pad_to_rank(shape, target)` / `pad_to_rank_4` / `try_strip_to_rank5` — single source of truth for the in-MLPackage rank convention. Used by every translator. |

### 3.3 Op classes with NO translator yet (gaps)

Surveyed across all canary probes. Each tract op below would unblock a specific model class:

| tract op | Where it surfaces | Estimated effort |
|---|---|---|
| `tract_core_scan` body recursion | DFN3 decoder, Parakeet decoder, Nemotron decoder | Architecturally invasive (~600 LoC pattern detector OR upstream tract change — see [`tract-rnn-preservation-handout.md`](./tract-rnn-preservation-handout.md)) |
| Detection-specific: `NonMaxSuppression`, `TopK`, `ScatterND`-with-NMS-shape | YOLOv8 with `nms=True` export, DETR-family | ~200 LoC each; not yet exercised by any canary |
| `Range` | Parakeet/Nemotron preprocessors (3× / 2× per model) | ~50 LoC; integer-only iota |
| `BitNot` | Parakeet/Nemotron preprocessors | trivial via `mb.logical_not` |
| `STFT` | Parakeet/Nemotron preprocessors | ~300 LoC; would unlock pure-audio preprocessing |
| `tract_core_bitnot`, `tract_core_eq` for TDim shape arithmetic | Parakeet/Nemotron preprocessors | Defer to upstream B5/B6 — these are TDim-shape ops that shouldn't reach a backend |

### 3.4 Op classes Apple-side has but we deliberately don't emit

| MIL op | Why we don't | Why we might revisit |
|---|---|---|
| `mb.scaled_dot_product_attention` | tract decomposes attention into `MatMul + Softmax + MatMul` — general_matmul handles it without needing the fused op | When the upstream RNN-preservation work (I16 — fused-contrib-op preservation) lands, the SDPA-fused form would benefit ANE engagement on transformers |
| `mb.gru`, `mb.lstm`, `mb.rnn` | tract surfaces RNN as `Scan(body=cell)` — pattern would have to detect cell-body shape | Same — see RNN preservation handout |
